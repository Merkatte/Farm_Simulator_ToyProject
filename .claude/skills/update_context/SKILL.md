# update_context 스킬

`AI_CONTEXT_Public/` 내 공용 문서 두 개를 최신 상태로 유지한다.

- **`log.md`**: 날짜별 작업 이력 (무엇을 했는지 간략 요약)
- **`Architecture.md`**: 현재 코드 구조와 파일 설명 (항상 최신 상태를 반영)

---

## 실행 절차

### [1단계] 현재 상태 파악

아래 파일들을 읽어 변경 사항을 파악한다.

1. `AI_CONTEXT_Public/log.md` — 마지막 기록 날짜와 내용 확인
2. `AI_CONTEXT_Public/Architecture.md` — 현재 문서 구조 확인
3. 이번 작업에서 생성·수정·삭제된 파일 목록 확인

### [2단계] log.md 업데이트

**규칙:**
- 오늘 날짜 섹션(`## YYYY-MM-DD`)이 이미 있으면 해당 섹션에 항목을 추가한다.
- 오늘 날짜 섹션이 없으면 파일 맨 아래에 새 섹션을 추가한다.
- 각 항목은 작업자(`### Claude` 또는 `### Codex`)를 명시한다.
- 내용은 **간결한 개조식**으로 작성한다. 무엇을 왜 했는지를 한두 줄로 요약한다.
- 코드, 긴 설명, 상세 구현 내용은 적지 않는다.

**포맷 예시:**
```markdown
## YYYY-MM-DD

### Claude

- `파일명` 수정 — 변경 이유 한 줄 요약
- `파일명` 신규 생성 — 목적 한 줄 요약
```

### [3단계] Architecture.md 업데이트

변경된 내용만 해당 섹션을 찾아 수정한다. 변경되지 않은 섹션은 건드리지 않는다.

**업데이트 대상 판단 기준:**

| 변경 유형 | 업데이트할 섹션 |
|----------|--------------|
| 스크립트 신규 추가 | 스크립트 목록에 항목 추가 |
| 스크립트 삭제 | 해당 항목 제거 |
| 스크립트 역할 변경 | 해당 항목 설명 수정 |
| 폴더 구조 변경 | 폴더 구조 다이어그램 수정 |
| 인터페이스/의존 관계 변경 | 의존 관계 다이어그램 수정 |
| 미구현 항목 구현됨 | "미구현 예정" 목록에서 제거 또는 완료 표시 |
| 새로운 미구현 항목 파악 | "미구현 예정" 목록에 추가 |

**규칙:**
- 파일 전체를 재작성하지 않는다. 변경이 필요한 부분만 수정한다.
- 실제 코드를 읽어 사실에 근거해 작성한다. 추측으로 쓰지 않는다.
- 코드 예시나 API 설명은 실제 현재 코드와 일치해야 한다.

### [4단계] 완료 보고

사용자에게 어떤 항목을 업데이트했는지 한두 줄로 보고한다.

## 제약 사항

- `AGENT.md`와 `.codex/` 폴더는 열지 않는다.
- `AI_CONTEXT_Codex/` 폴더는 열지 않는다.
- `AI_CONTEXT_Claude/`의 내부 로그는 참고용으로만 읽고, 해당 폴더에는 쓰지 않는다.
- `log.md`에는 상세 코드나 구현 설명을 쓰지 않는다. 요약만 한다.
- `Architecture.md`에는 추측성 내용을 쓰지 않는다. 현재 코드 기준으로만 작성한다.
